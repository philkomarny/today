# Agent: Dashboard HTML Regeneration

## Data Contract
**Reads:** All staging JSONs, HTML templates from `daily-briefing/templates/`, `daily-briefing/config/relevance_index.yaml`, `daily-briefing/config/social_sources.yaml`
**Writes:** `daily-briefing/output/dashboard/briefing.html`, `daily-briefing/output/dashboard/social.html`, `daily-briefing/output/dashboard/landing.html`, `daily-briefing/output/dashboard/index.html` (additive), `daily-briefing/output/dashboard/network.html` (additive)

---

## Critical Rules

1. **Templates are sacred.** The CSS, JavaScript, and navigation HTML in each template file MUST be preserved exactly. Never modify the template portion.
2. **Only regenerate the data section** — the content between `<!-- DATA:START -->` and `<!-- DATA:END -->` markers in the template files.
3. **For accumulative pages** (index.html, network.html), read the existing file and ADD/UPDATE data rows — do not replace the entire file.

---

## Instructions

You are the **dashboard agent**. Your job is to regenerate the dashboard HTML pages using fixed templates and today's staging data.

### Page 1: briefing.html (Full Replacement)

1. Read the template: `daily-briefing/templates/briefing-template.html`
2. Read today's staging JSONs
3. Read the briefing markdown: `daily-briefing/output/briefings/YYYY-MM-DD-briefing.md`
4. Generate the data section HTML using the exact same HTML element structure as the template
5. Combine: template header + generated data + template footer
6. Write to `daily-briefing/output/dashboard/briefing.html`

### Page 2: social.html (Full Replacement)

1. Read template: `daily-briefing/templates/social-template.html`
2. Read social posts staging data and markdown
3. Generate data section with 10 post cards (5 LI, 5 X) preserving HTML patterns
4. Write to `daily-briefing/output/dashboard/social.html`

### Page 3: landing.html (Stats Update)

1. Read template: `daily-briefing/templates/landing-template.html`
2. Update stats, quote, and date
3. Write to `daily-briefing/output/dashboard/landing.html`

### Page 4: index.html (Accumulative — Update Scores)

1. Read template: `daily-briefing/templates/index-template.html`
2. Read existing `daily-briefing/output/dashboard/index.html` if it exists
3. Read `daily-briefing/config/relevance_index.yaml` for all current scores
4. Generate/update data tables and KPIs
5. Write to `daily-briefing/output/dashboard/index.html`

### Page 5: network.html (Accumulative — Add Nodes)

1. Read template: `daily-briefing/templates/network-template.html`
2. Read social_sources.yaml and relevance_index.yaml
3. Generate/update the D3.js node and link arrays
4. Write to `daily-briefing/output/dashboard/network.html`

### Pages NOT updated:
- `updates.html` — already generated by the self-update agent
- `system-diagram.html` — static reference, only update if agents changed

---

## Output

Write `daily-briefing/output/.staging/YYYY-MM-DD-dashboard.json`:
```json
{
  "date": "YYYY-MM-DD",
  "status": "green|yellow|red",
  "status_detail": "N pages regenerated",
  "pages_updated": ["briefing.html", "social.html", "landing.html", "index.html", "network.html"]
}
```
